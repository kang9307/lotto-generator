서브넷(Subnet)이란?

서브넷은 IP 네트워크의 논리적 분할입니다. 서브넷팅을 통해 하나의 큰 네트워크를 여러 개의 작은 네트워크로 나눌 수 있으며, 이는 네트워크 관리와 효율성을 향상시킵니다.

주요 용어

  • IP 주소: 인터넷 프로토콜에 사용되는 주소로, 장치를 식별하는 고유 번호입니다.
  • 서브넷 마스크: IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 값입니다.
  • CIDR (Classless Inter-Domain Routing): IP 주소 할당 방법으로, /24와 같은 접두사 길이로 표현됩니다.
  • 네트워크 주소: 해당 서브넷의 첫 번째 주소로, 호스트 부분이 모두 0인 주소입니다.
  • 브로드캐스트 주소: 해당 서브넷의 마지막 주소로, 호스트 부분이 모두 1인 주소입니다.

서브넷 계산기 사용법

  1. IP 주소 입력 필드에 유효한 IPv4 주소를 입력합니다 (예: 192.168.1.0).
  2. CIDR 표기법 또는 서브넷 마스크 중 하나를 선택하여 입력합니다:
    • CIDR: 드롭다운 목록에서 적절한 프리픽스 길이를 선택합니다 (예: /24).
    • 서브넷 마스크: 유효한 서브넷 마스크를 입력합니다 (예: 255.255.255.0).
  3. "계산하기" 버튼을 클릭하여 결과를 확인합니다.

계산 결과 해석

  • 네트워크 주소: 해당 서브넷의 시작 주소입니다.
  • 브로드캐스트 주소: 해당 서브넷의 마지막 주소입니다.
  • 호스트 수: 네트워크 내에서 사용 가능한 IP 주소의 수입니다 (네트워크 주소와 브로드캐스트 주소 제외).
  • 호스트 범위: 실제로 장치에 할당할 수 있는 IP 주소의 범위입니다.
  • 바이너리 표현: IP 주소와 서브넷 마스크의 2진수 표현입니다.
