# 블로그 포스트 변환 가이드

이 문서는 블로그 포스트를 독립적인 HTML 파일로 변환하는 방법을 설명합니다.

## 개요

블로그 포스트를 다음과 같은 이유로 독립적인 HTML 파일로 변환합니다:

1. 구글 인덱싱 최적화 (URL 파라미터 방식은 검색 엔진 인덱싱에 문제가 있음)
2. 각 포스트를 사이트의 다른 페이지와 같은 형식으로 통일
3. 헤더와 푸터를 포함한 일관된 사용자 경험 제공

## 변환 스크립트 사용 방법

### 1. 준비 사항

- Node.js가 설치되어 있어야 합니다 (v12 이상 권장).

### 2. 스크립트 실행

다음 명령어로 변환 스크립트를 실행합니다:

```bash
node convert_posts.js
```

### 3. 실행 결과

- 스크립트는 `posts` 디렉토리에서 모든 HTML 및 마크다운 파일을 찾습니다.
- 각 파일을 템플릿을 적용한 독립 HTML 파일로 변환합니다.
- 결과 파일은 `posts` 디렉토리에 저장됩니다.
- sitemap.xml 파일이 자동으로 업데이트됩니다.

## 디렉토리 구조

변환 후 디렉토리 구조는 다음과 같습니다:

```
lotto-generator/
  ├── blog.html          # 블로그 메인 페이지
  ├── blog.js            # 블로그 기능 스크립트
  ├── common.js          # 공통 스크립트
  ├── convert_posts.js   # 변환 스크립트
  ├── post_template.html # HTML 템플릿
  ├── posts/             # 원본 및 변환된 포스트
  │   ├── post1.md       # 원본 마크다운 파일
  │   ├── post1.html     # 변환된 HTML 파일
  │   ├── post2.html     # 원본 HTML 파일
  │   └── post2.html     # 변환된 HTML 파일 (덮어쓰기)
  ├── sitemap.xml        # 업데이트된 사이트맵
  └── styles.css         # 스타일시트
```

## 문제 해결

1. **변환 오류 발생 시**: 오류 메시지를 확인하고 해당 파일의 구조나 내용을 확인합니다.
2. **메타데이터 추출 실패**: 포스트 파일에 필요한 메타데이터가 올바르게 포함되어 있는지 확인합니다.
3. **HTML 템플릿 문제**: `post_template.html` 파일이 올바른 형식으로 되어 있는지 확인합니다.

## 유지 관리

새 포스트를 추가할 때마다 다음 단계를 수행합니다:

1. 새 포스트를 `posts` 디렉토리에 추가합니다.
2. 변환 스크립트를 실행합니다.
3. 생성된 HTML 파일을 확인합니다.
4. 업데이트된 sitemap.xml 파일을 확인합니다.

## 참고 사항

- 이 스크립트는 이미 존재하는 HTML 파일을 덮어씁니다.
- 원본 포스트 파일은 그대로 유지됩니다.
- 마크다운 파일은 HTML로 변환되어 저장됩니다.
